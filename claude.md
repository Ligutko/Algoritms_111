## Визначення алгоритму

### 🎯 Основне визначення

**Алгоритм** — це впорядкована послідовність кроків, що однозначно визначають процес автоматичного розв'язання обчислювальної задачі.

### 📝 Альтернативні визначення

1. Послідовність математичних та логічних операцій, що ведуть до рішення класу задач за скінченне число кроків

2. Послідовність дій для обробки початкових даних з метою отримання результатів

3. Заздалегідь задане, зрозуміле та точне розпорядження, що визначає послідовність операцій для отримання рішення за кінцеве число кроків

4. **Позначення дій** — називаються **командами** або **інструкціями** (statement)

---

## Основні властивості

### ✅ Зрозумілість (Clarity)
Виконавець алгоритму повинен мати можливість зрозуміти та правильно виконати кожну команду

### ✅ Детермінованість (Determinacy)
- Кожна команда має бути чіткою та однозначною
- Не допускаються різні тлумачення команд

### ✅ Дискретність (Discreteness)
- Алгоритм складається з окремих, чітко визначених кроків
- Кроки чітко відокремлені один від одного

### ✅ Завершеність та Результативність (Termination)
- Алгоритм обов'язково повинен завершити роботу за скінченний час
- Повинен видати результат

### ✅ Масовість (Universality)
- Можливість застосування до цілого класу однотипних задач
- Має працювати з різними наборами вхідних даних

### ✅ Виконуваність (Feasibility)
- Алгоритм завжди можна виконати до кінця
- Не повинен обриватися без отримання результату

## Правила побудови

### 🔧 Етапи побудови

#### 1️⃣ Визначення об'єктів
- Задати об'єкти, з якими працюватиме алгоритм
- Визначити їх типи та властивості

#### 2️⃣ Вхідні та вихідні дані
- **Вхідні дані**: набір даних для обробки
- **Вихідні дані**: результати роботи алгоритму

#### 3️⃣ Пам'ять
Мають бути передбачені місця для:
- Зберігання вхідних даних
- Збереження проміжних результатів
- Розміщення вихідних даних

#### 4️⃣ Дискретність
Алгоритм розбивається на окремі дії (операції, команди)

#### 5️⃣ Детермінованість
- Після кожного кроку вказується наступний
- Або видається команда завершення

#### 6️⃣ Збіжність (Результативність)
- Завершення після кінцевого числа кроків
- Явне вказання результату

## Способи опису

### 📖 1. Словесно-формульний
- Описання натуральною мовою
- З елементами математичної символіки
- Менш точний, але легший для розуміння

### 📊 2. Графічний (Блок-схема)
- Стандартні символи для операцій
- Показує послідовність кроків
- Графічне представлення потоку виконання

### 💻 3. Псевдокодом
- Гібрид мови програмування та природної мови
- Детальніше за словесний опис
- Абстрагується від конкретної мови

### ⚙️ 4. Програмний
- Запис мовою програмування
- Найбільш конкретний та точний
- Готовий до виконання

## Типові структури

### 🔹 1. Лінійна структура
```
Дія 1
Дія 2
Дія 3
...
```
- Операції виконуються послідовно
- Немає умовних переходів чи циклів
- Проста для розуміння та реалізації

### 🔹 2. Розгалужена структура
```
IF умова THEN
    Гілка 1
ELSE
    Гілка 2
ENDIF
```
- Залежить від умови
- **Простий розгалік**: два напрямки
- **Складний розгалік**: більше двох напрямків

### 🔹 3. Циклічна структура
```
WHILE умова DO
    Тіло циклу
ENDWHILE
```
- Повторення блоків кода
- Один або більше циклічних блоків
- Важлива для багаторазових операцій

## Часова складність

### ⏱️ Визначення часу виконання

**T(n)** — кількість елементарних операцій, виконаних комп'ютером

**Елементарні операції:**
- Присвоєння значення
- Арифметичні операції (+, -, *, /)
- Операції порівняння (<, >, =)
- Виклик функції
- Індексація масиву

### 📈 Три типи часової складності

#### 1. **Найгірший випадок (Worst-case)**
- Максимальна кількість операцій
- Найпесимістичніша оцінка
- **На практиці використовується найчастіше**

#### 2. **Найкращий випадок (Best-case)**
- Мінімальна кількість операцій
- Оптимістична оцінка
- Рідко використовується на практиці

#### 3. **Середній випадок (Average-case)**
- Усереднена кількість операцій
- Складніша для розрахунку
- Потребує ймовірнісних методів

### 🎯 Асимптотична поведінка

При великих значеннях n цікавит **порядок** складності, а не точні значення.

**Правила спрощення:**
1. Опускаються константні множники
2. Враховується лише найшвидше зростаючий доданок

**Приклад:**
$$T(n) = 345n^3 + 123n^2 + 98n + 15 \approx O(n^3)$$

## Асимптотичні нотації

### 🔤 Theta-нотація: Θ(n)

Точна оцінка складності — функція обмежена як зверху, так і знизу.

$$\Theta(g(n)) = \{f(n) : \exists c_1, c_2, n_0 > 0 \text{ | } c_1g(n) \leq f(n) \leq c_2g(n) \forall n \geq n_0\}$$

**Значення:** $f(n) = \Theta(g(n))$ означає, що функції мають однаковий порядок росту.

### 🔤 Big-O-нотація: O(n)

Верхня межа складності — алгоритм виконується не гірше ніж O(n).

$$O(g(n)) = \{f(n) : \exists c, n_0 > 0 \text{ | } f(n) \leq c \cdot g(n) \forall n \geq n_0\}$$

**Використання:** Найпоширеніша нотація для оцінки худшого випадку.

### 🔤 Omega-нотація: Ω(n)

Нижня межа складності — алгоритм виконується не швидше ніж Ω(n).

$$\Omega(g(n)) = \{f(n) : \exists c, n_0 > 0 \text{ | } c \cdot g(n) \leq f(n) \forall n \geq n_0\}$$

**Взаємозв'язок:**
$$f(n) = \Theta(g(n)) \iff f(n) = O(g(n)) \text{ AND } f(n) = \Omega(g(n))$$

---

## Правила O-нотації

### 📐 Математичні властивості

1. **Опускання констант**
   - $C \cdot n^3 = O(n^3)$ для будь-якої константи C

2. **Порівняння експонент**
   - Якщо $b \geq a$, то $n^a = O(n^b)$

3. **Експоненціали переважають поліноми**
   - $n^k = O(a^n)$ для $a > 1$

4. **Логарифми ростуть повільне**
   - $\log^k n = O(n)$ для будь-якого $k$

5. **Домінуючий доданок**
   - Складність визначається найшвидше зростаючим доданком

**Приклад:**
$$345n^3 + 123n^2 + 98n + 15 = O(n^3)$$

## Поширені складності

### 📊 Таблиця складностей

| O-нотація | Назва | Характеристика | Приклади |
|-----------|-------|---|---|
| **O(1)** | Константна | Незалежна від розміру | Доступ до елемента, простої операції |
| **O(log n)** | Логарифмічна | Уповільнене зростання | Бінарний пошук |
| **O(n)** | Лінійна | Пропорційна розміру | Лінійний пошук, простий цикл |
| **O(n log n)** | Лінійно-логарифмічна | Ефективне сортування | Merge Sort, Quick Sort |
| **O(n²)** | Квадратична | Вложені цикли | Bubble Sort, Selection Sort |
| **O(n³)** | Кубічна | Потрійні цикли | Матричне множення |
| **O(2ⁿ)** | Експоненціальна | Дуже повільна | Переченьлення всіх підмножин |
| **O(n!)** | Факторіальна | Найповільніша | Перестановки всіх елементів |

### 📈 Графічне порівняння

```
Час виконання
     ↑
     |     n! (факторіальна)
     |    /
     |   / 2ⁿ (експоненціальна)
     |  /
     | / n³ (кубічна)
     |/  n² (квадратична)
────┼───────────→ (розмір вхідних даних)
     |\ n log n
     | \n (лінійна)
     |  \log n (логарифмічна)
     |   \1 (константна)
```

## Правила оцінки

### 🧮 Послідовні оператори

Для послідовності $S_1 S_2 ... S_m$:
$$\text{T(n)} = O(\max\{T_1(n), T_2(n), ..., T_m(n)\})$$

### 🔁 Цикл While

```
while (умова):
    S1
    S2
```

$$\text{T(n)} = O(\max\{T_1(n) × (I(n) + 1), T_2(n) × I(n)\})$$

Де $I(n)$ — кількість ітерацій в гіршому випадку.

### 🔁 Цикл For

```
for i in range(n):
    S1
    S2
```

$$\text{T(n)} = O(\max\{T_1(n) × n, T_2(n) × n\})$$

### 🔀 Умовний оператор

```
if умова:
    S1
else:
    S2
else if:
    S3
```

$$\text{T(n)} = O(\max\{T_1(n), T_2(n), T_3(n)\})$$

## Практичні приклади

### 💡 Завдання 1: O(1)

```python
def get_first_element(array):
    return array[0]
```

**Аналіз:** Операція не залежить від розміру array.  
**Складність:** ✅ **O(1)**

---

### 💡 Завдання 2: O(n)

```python
def sum_array(array):
    sum = 0
    for i in range(len(array)):
        sum += array[i]
    return sum
```

**Аналіз:** 
- Цикл виконується n разів
- Кожна операція всередину циклу — O(1)
- Загалом: n × O(1)

**Складність:** ✅ **O(n)**

---

### 💡 Завдання 3: O(log n)

```python
def divide_by_three(n):
    i = n
    while i > 0:
        print(i)
        i = i // 3
```

**Аналіз:**
- На кожній ітерації i ділиться на 3
- Це повторюється log₃(n) разів
- Властивості логарифмів дозволяють опустити основу

**Складність:** ✅ **O(log n)**

---

### 💡 Завдання 4: O(n) часова, O(n) пам'яті

```python
def create_doubled_array(n):
    array = [0] * n  # Виділення пам'яті
    for i in range(n):
        array[i] = i * 2
    return array
```

**Аналіз:**
- **Часова:** Цикл виконується n разів → **O(n)**
- **Пам'ять:** Створюється масив розміром n → **O(n)**

---

### 💡 Завдання 5: O(n²) часова, O(1) пам'яті

```python
def print_pairs(n):
    for i in range(n):
        for j in range(n):
            print(i, j)
```

**Аналіз:**
- **Часова:** Вложені цикли n × n = **O(n²)**
- **Пам'ять:** Тільки змінні i, j → **O(1)**

## Важливість вибору алгоритму

### 💰 Вплив алгоритму на продуктивність

При одиниці часу = 1 мілісекунда, максимальний розмір завдання за 1 хвилину:

| Складність | A1: O(1) | A2: O(log n) | A3: O(n) | A4: O(n log n) | A5: O(n²) | A6: O(2ⁿ) |
|--|--|--|--|--|--|--|
| n (базовий) | ∞ | 2M | 60K | ~6K | ~1K | 20 |
| Після 10× прискорення | ∞ | 2M | 600K | ~60K | ~3.2K | 23 |
| Зростання | ×1 | ×1 | ×10 | ×10 | **×3.2** | **×1.5** |

### 🎯 Висновки з таблиці

✅ **Алгоритм O(n) в 10 разів швидший** при 10× прискоренні

✅ **Алгоритм O(n²) дає лише 3.2× поліпшення**

✅ **Алгоритм O(2ⁿ) практично не поліпшується**

**Ключова ідея:** Вибір ефективнішого алгоритму часто важливіше, ніж прискорення обладнання!
