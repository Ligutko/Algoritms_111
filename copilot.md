# Алгоритми та їх властивості

## Визначення алгоритму

Алгоритм – це **логічна, впорядкована послідовність кроків виконання для досягнення цільового результату**.

### Основні визначення:

- **Алгоритм** — послідовність математичних, логічних або разом узятих операцій, що відрізняються детермінованістю, масовістю, спрямованістю і приводить до рішення всіх завдань даного класу за кінцеве число кроків

- **Алгоритм** — послідовність дій, які потрібно виконати для обробки сукупності початкових даних і здобуття відповідних результатів

- **Алгоритм** — заздалегідь задане зрозуміле і точне розпорядження можливому виконавцеві виконати певну послідовність дій для отримання рішення задачі за кінцеве число кроків

- **Позначення дій** в алгоритмі називаються **командами** або **інструкціями**

---

## Основні риси (властивості) алгоритму

### 1. **Зрозумілість**
- Виконавець алгоритму може правильно зрозуміти та виконати команди, записані в алгоритмі

### 2. **Детермінованість (однозначність)**
- Кожна команда в алгоритмі має бути чіткою й однозначною

### 3. **Дискретність (перервність, роздільність)**
- Алгоритм задає послідовність дій, чітко відокремлених одна від одної

### 4. **Завершеність та результативність**
- Обов'язково здобуття результату після реалізації алгоритму за скінченний час

### 5. **Масовість**
- Можливість застосування певного алгоритму до цілого класу однотипних задач
- Алгоритм має використовуватися для різних вихідних даних

### 6. **Виконуваність (скінченність)**
- Алгоритм завжди можна виконати до кінця
- Процес, заданий алгоритмом, має тривати скінченний час і не повинен обриватися без отримання результату

---

## Правила побудови алгоритму

1. **Об'єкти**: Необхідно задати об'єкти, з якими буде працювати алгоритм

2. **Вхідні та вихідні дані**:
   - Алгоритм приступає до роботи з деяким набором даних — **вхідних даних**
   - У результаті видає дані — **вихідні дані**

3. **Память**: Для роботи алгоритму потрібна пам'ять для розміщення:
   - Вхідних даних
   - Проміжних даних
   - Вихідних даних (результатів)

4. **Дискретність**: Алгоритм будується з окремих кроків (дій, операцій, команд)

5. **Детермінованість**: Після кожного кроку необхідно вказувати, який крок виконується наступний, або давати команду зупинки

6. **Збіжність (результативність)**: Алгоритм повинен завершувати роботу після кінцевого числа кроків, з явним вказанням результату

---

## Способи опису алгоритмів

### 1. **Словесно-формульний**
Опис засобами природної мови з елементами математичної символіки

### 2. **Графічний (блок-схема)**
- Графічне зображення, яке демонструє послідовність основних кроків виконання програми
- Використовуються стандартні символи для різних операцій

### 3. **Псевдокодом**
- Композиція конструкцій мови програмування та узагальнених операторів на простій мові
- Детальніше, ніж словесно-формульне описання

### 4. **Програмний**
- Запис алгоритму мовою програмування

---

## Типові структури алгоритму

### 1. **Лінійна структура**
- Операції виконуються послідовно, почерзі їх запису
- Немає розгалужень або повторень

### 2. **Розгалужена структура**
- Існує умова, залежно від виконання якої є кілька напрямків обчислень
- **Простий розгалужений алгоритм**: два напрямки
- **Складний розгалужений алгоритм**: більше двох напрямків

### 3. **Циклічна структура**
- Процес з одним або більше блоками, що повторюються

---

## Покрокове проектування алгоритмів

### Етап 1: Математична модель
- Створення моделі вихідної задачі
- Застосування відповідних математичних моделей
- Побудова неформального алгоритму для знаходження рішення

### Етап 2: Псевдокод
- Запис алгоритму на псевдомові
- Заміна неформальних операторів послідовністю детальних операторів
- Фіксування різних типів даних, над якими виконуються оператори

### Етап 3: Абстрактні типи даних
- Створення абстрактних типів даних для кожного фіксованого типу
- Завдання імен функцій для кожного оператора
- Заміна операторів викликом відповідних функцій

### Етап 4: Програмування
- Реалізація кожного абстрактного типу даних
- Створення функцій для виконання операторів
- Заміна всіх неформальних операторів кодом мови програмування
- Наладжування отриманої програми

## Аналіз алгоритму

### Основні питання:

- **Поведінка**: Знаходить ли алгоритм найкраще можливе або просто не гірше рішення? Є чи кілька найкращих рішень?

- **Швидкість**: Швидкий ли алгоритм? Уповільнює ли роботу тільки з деякими вхідними даними?

- **Вимоги до пам'яті**: Скільки комп'ютерних ресурсів йому необхідно? Чи є такий обсяг прийнятним?

- **Методи**: Можна ли задіяти методи повторно для вирішення подібних завдань?

### Критерії аналізу:

✓ Час роботи програми як функція від вхідних даних  
✓ Загальна кількість пам'яті, необхідна для програми  
✓ Загальний об'єм програмного коду  
✓ Чи програма розв'язує коректно поставлену задачу  
✓ Чи є програма стресо-стійкою (поведінка з некоректними даними)  
✓ Складність програми (легкість читання, розуміння, модифікації)

---

## Час виконання алгоритму

### Як оцінити алгоритм?

- **Як оцінити швидкодію?**
- **У яких одиницях вимірювати швидкодію?**
- **Яка буде складність алгоритму?**

### Елементарні операції

Робота будь-якої програми складається з елементарних операцій:
- Присвоєння
- Елементарні арифметичні операції
- Операції порівняння
- Виклик функції/методу
- Звернення за індексом до елементу списку (масиву)

**Час виконання програми** — кількість елементарних операцій, які виконує комп'ютер під час виконання програми.

### Позначення часової складності

**T(n)** — час виконання програми, де **n** — розмір вхідних даних

Часова складність — функція часу від вхідних даних, що відображає залежність кількості операцій від розміру вхідних даних.

### Три типи часової складності:

1. **Найгірший випадок (worst-case running time)**
   - Найбільша кількість елементарних операцій для довільних вхідних даних розміру n

2. **Найкращий випадок (best-case running time)**
   - Найменша кількість елементарних операцій для деякого набору вхідних даних розміру n

3. **Середній випадок (average-case running time)**
   - Усереднена кількість елементарних операцій для всіх наборів вхідних даних розміру n

**На практиці:** Час виконання в найгіршому випадку використовується як характеристика часової складності алгоритму, оскільки визначення середнього часу значно складніше.

---

## Складність алгоритму

**Складність алгоритму** — кількісна характеристика, що відображає спожирані алгоритмом ресурси під час виконання.

### Два типи складності:

1. **Часова складність**
   - Показник ефективності алгоритму з точки зору часу виконання

2. **Складність по пам'яті**
   - Показник ефективності алгоритму з точки зору використовуваної пам'яті

---

## Часова складність алгоритму

### Приклад з великими вхідними даними

При оцінці швидкодії алгоритму найбільше цікавить поведінка **T(n)** при великому розмірі вхідних даних.

- Цікавить не точна кількість операцій
- А **порядок** кількості операцій

### Правила оцінки складності:

1. **У формулі T(n)**, враховуються лише доданки, що зростають найшвидше
2. **Сталий множник** при цьому доданку встановлюється рівним 1

### Приклад:

Для алгоритму з часом виконання:
$$T(n) = 345n^3 + 123n^2 + 98n + 15$$

**Порядок складності буде:** $n^3$

### Асимптотична поведінка

Асимптотична складність — це лімітна поведінка функції часу виконання при $n \to \infty$

- Цікавить, як час роботи алгоритму зростає з збільшенням розміру вхідних даних
- Розглядаємо поведінку при граничному збільшенні розміру

---

## Складність алгоритму по пам'яті

Аналіз складності по пам'яті проводиться аналогічно часовій складності:
- Замість кількості операцій аналізується пам'ять, яку використовує програма
- Оцінюється **максимальна величина пам'яті**, виділена під час виконання алгоритму

---

## Асимптотичні нотації

### **Θ-нотація (Theta notation)**

Для заданої функції $g(n)$, запис $\Theta(g(n))$ означає множину функцій, такої що:

Функція $f(n)$ належить множині $\Theta(g(n))$, якщо існують додатні константи $c_1$ та $c_2$, які дозволяють заключити цю функцію в межі між функціями $c_1g(n)$ та $c_2g(n)$ для достатньово великих $n$.

**Говорять:** функція $g(n)$ є **асимптотично точною оцінкою** функції $f(n)$.

### **O-нотація (Big-O notation)**

О-нотація визначає функцію $f(n)$, яка показує, як буде змінюватися складність алгоритму при змінці кількості вхідних даних у найгіршому для алгоритму випадку.

**Визначення:** Говорять, що функція часу $T(n)$ має порядок росту $O(f(n))$, якщо існують константи $c$ і $n_0$ такі, що для усіх $n \geq n_0$ виконується нерівність:

$$T(n) \leq c \cdot f(n)$$

**Примітка:** Якщо $f(n) = \Theta(g(n))$, то $f(n) = O(g(n))$, оскільки Θ-позначення є більш сильне.

### **Ω-нотація (Omega notation)**

Аналогічно O-позначенням, які дають асимптотичну верхню границю, Ω-позначення дає асимптотичну нижню границю функції $g(n)$.

**Співвідношення:**
$$f(n) = \Theta(g(n)) \text{ тоді й тільки тоді, коли } f(n) = O(g(n)) \text{ та } f(n) = \Omega(g(n))$$

---

## Правила використання O-нотації

### Властивості асимптотичної складності:

1. **Мультиплікативні константи можна опускати**
   - Для будь-якої додатної сталої C: $C \cdot n^3 = O(n^3)$

2. **Степені порівняння:** Якщо $b \geq a$, то $n^a = O(n^b)$

3. **Експоненціали переважають поліноми:** Якщо $a > 1$, то для будь-якого $k \geq 1$: $n^k = O(a^n)$

4. **Логарифми росту повільно:** Для будь-якого $k \geq 1$: $\log^k n = O(n)$

5. **Домінуючий доданок:** Якщо функція є сумою кілька функцій, то її асимптотична поведінка визначається доданком, що зростає найшвидше.
   
   Приклад: $345n^3 + 123n^2 + 98n + 15 = O(n^3)$

---

## Поширені асимптотичні складності

| Нотація | Назва | Опис |
|---------|-------|------|
| O(1) | Константна | Час виконання не залежить від розміру даних |
| O(log n) | Логарифмічна | Час зростає логарифмічно (пошук у відсортованому масиві) |
| O(n) | Лінійна | Час зростає лінійно зі збільшенням даних |
| O(n log n) | Лінійно-логарифмічна | Ефективні алгоритми сортування |
| O(n²) | Квадратична | Простіші алгоритми сортування |
| O(n³) | Кубічна | Алгоритми для матриці 3x3 |
| O(2ⁿ) | Експоненціальна | Дуже повільна для великих даних |
| O(n!) | Факторіальна | Найповільніша, практично непридатна |

## Правила оцінки алгоритмів

### Послідовні оператори
Найгірший час виконання алгоритму, що складається з послідовності виразів $S_1 S_2 ... S_m$ має асимптотичну складність:

$$O(\max\{T_1(n), T_2(n), ..., T_m(n)\})$$

### Цикл while
$$O(\max\{T_1(n) \times (I(n) + 1), T_2(n) \times I(n)\})$$

де $T_1(n), T_2(n)$ — час виконання інструкцій, $I(n)$ — кількість ітерацій у найгіршому випадку

### Цикл for
$$O(\max\{T_1(n) \times (I(n) + 1), T_2(n) \times I(n)\})$$

### Умовний оператор
$$O(\max\{T_1(n), T_2(n), T_3(n)\})$$

де $T_1(n), T_2(n), T_3(n)$ — час виконання трьох гілок

## Практичні приклади визначення складності

### Завдання 1: O(1) – Константна складність

```
result = array[0]
print(result)
```

**Розв'язок:** При збільшенні вхідного параметра n кількість операцій не змінюється. Асимптотична складність — **O(1)**.

---

### Завдання 2: O(n) – Лінійна складність

```
sum = 0
for i in range(n):
    sum += array[i]
print(sum)
```

**Розв'язок:** Кількість операцій залежить від n. При збільшенні n в 10 разів кількість операцій збільшиться не більш ніж в 10 разів. Асимптотична складність — **O(n)**.

---

### Завдання 3: O(log n) – Логарифмічна складність

```
i = n
while i > 0:
    print(i)
    i = i // 3
```

**Розв'язок:** На кожній ітерації цикл ділить i на 3. Цикл виконується log₃(n) разів.

Використані властивості:
- $\log_a n = c \cdot \log_b n$ (логарифми з різними основами пропорційні)
- Константи в О-нотації можна опускати

Асимптотична складність: **O(log n)**

---

### Завдання 4: O(n) за часом і пам'яттю

```
array = [0] * n
for i in range(n):
    array[i] = i * 2
```

**Розв'язок:**
- **Часова складність:** Цикл має n ітерацій → **O(n)**
- **Складність по пам'яті:** Створюється масив розміром n → **O(n)**

---

### Завдання 5: O(n²) за часом, O(1) по пам'яті

```
for i in range(n):
    for j in range(n):
        print(i * j)
```

**Розв'язок:**
- **Часова складність:** Вложені цикли, кожен з n ітерацій → n × n = **O(n²)**
- **Складність по пам'яті:** Використовується тільки три змінні (i, j, результат) → **O(1)**
